draft — started 2026-03-05

There's something about watching a build process churn through thousands of files that teaches you what no tutorial can. Patience isn't passive — it's the active decision to keep iterating when the tests are red.

The compiler doesn't care about your deadlines. It processes when it's ready. And yet, every morning you show up, you write the code, you fix the bugs. That consistency isn't glamorous, but it compiles into something real.

I think writing is the same. You push commits of thought, you refactor understanding, and sometimes — if the logic is sound — something compiles that surprises even you.

design_rationale

color_scheme

Ultra-dark background (#0a0e14) with phosphor green (#39ff14) as primary accent. Cyan for tags, amber for filenames/strings. Inspired by classic terminal themes.

typography

JetBrains Mono for everything (monospace continuity). Space Grotesk for titles and readable prose blocks. Creates a "developer's notebook" vibe.

layout_shift

Replaces card grid with a flat list (like ls output). Entries show as table rows with date, title, tags, metadata. Minimal decoration — content-first.

editor_changes

Full IDE-inspired editor with line numbers, vim-style commands (:w, :publish), status bar at bottom, markdown-syntax toolbar buttons, and file path breadcrumb.

key_departure

Strips all ornamental design. No glowing runes, no hover animations. Information density over decoration. Every element has a functional purpose.

interaction

Hover states are subtle (slight brightness shift). Keyboard shortcuts emphasized. Status bar provides constant feedback. Green glow is reserved for active/selected states only.
